# Компонент «Level Happiness» (`dreamteam:level.happiness`)

Подробное описание локального компонента 1С-Битрикс для опроса «уровня счастья» сотрудников: сбор индивидуальных оценок, агрегированная статистика и обновление интерфейса без перезагрузки страницы.

---

## 1. Назначение

Виджет позволяет авторизованному пользователю выбрать уровень удовлетворённости (шкала от `MIN_LEVEL` до `MAX_LEVEL`, по умолчанию 1–5 «звёзд»), сохранить выбор и увидеть:

- среднюю оценку по всем голосам;
- количество голосов;
- распределение голосов по уровням (полосы процентов и счётчики);
- визуализацию среднего значения в виде звёзд (полная / половина / пустая);
- свой текущий уровень (если уже голосовал).

Тексты формы, цвета столбиков, ссылки «написать руководству» и прочие строки настраиваются через `config.php` и частично через параметры компонента в визуальном редакторе.

---

## 2. Технологический стек

| Слой | Технологии |
|------|------------|
| CMS | 1С-Битрикс (`B_PROLOG_INCLUDED`, `CBitrixComponent`, `CUtil`, `bitrix_sessid_post`, `BX.ajax.runComponentAction`) |
| Сервер | PHP 7.4+ (предположительно; используются типы возврата `void`, классы ядра D7) |
| Хранилище | JSON-файл на диске (не БД) |
| Фронтенд | Vanilla JS (IIFE), API `BX.ajax.runComponentAction`, опционально `BX.UI.Notification` |
| Стили | CSS с переменными `:root`, адаптивный блок `@media (max-width: 400px)` |

Компонент реализует интерфейс `Controllerable` и регистрирует AJAX-действие `saveLevel` для сохранения оценки.

## 3. Структура каталога

```
level.happiness/
├── .description.php          — метаданные для списка компонентов в админке
├── .parameters.php           — параметры компонента (редактор шаблонов)
├── .section.php              — служебное имя секции Bitrix
├── component.php             — точка входа: подключение класса и executeComponent()
├── class.php                 — основная логика: данные, статистика, AJAX, уведомления
├── config.php                — централизованные настройки и тексты по умолчанию
├── init.php                  — автоподключение виджета через $APPLICATION->IncludeComponent
├── data/
│   ├── .section.php
│   └── levelHappiness.json   — данные: объект USER { "userId": level, ... }
└── templates/
    ├── .section.php
    └── .default/
        ├── .section.php
        ├── template.php      — разметка виджета и вывод FRONTEND_CONFIG в JS
        ├── script.js         — отправка формы, обновление DOM, перенос виджета
        └── style.css         — оформление карточки, звёзд, полос, формы
```

Шаблон по факту один — `.default`. В `config.php` указан `WIDGET_STYLE` со значениями `custom` | `bitrix`; `init.php` ищет папку шаблона с таким именем и при отсутствии откатывается на `custom` или `.default`. В репозитории присутствует только `.default`, поэтому для стиля `custom` без дублирования папки фактически используется `.default`.

## 4. Формат данных (`levelHappiness.json`)

Файл — валидный JSON с корневым объектом и ключом `USER`:

```json
{
  "USER": {
    "123": 5,
    "456": 3
  }
}
```

- Ключи — строковые идентификаторы пользователей Bitrix.
- Значения — целые уровни в допустимом диапазоне.
- При чтении некорректный или пустой файл трактуется как `{ "USER": {} }`.

Запись выполняется атомарно: содержимое пишется во временный файл `*.tmp` с блокировкой `LOCK_EX`, затем выполняется `rename` поверх целевого файла (с запасным вариантом через `unlink` для окружений, где перезапись `rename` недоступна).

## 5. Логика на сервере (`class.php`)

### 5.1. `executeComponent()`

1. Читает JSON (`readData()`).
2. Нормализует голоса (`normalizeVotes()`).
3. Строит статистику `buildStats()` для текущего пользователя:
   - `stars.STATIC[level]` — число голосов за уровень;
   - `stars.GLAVCOUNT` — всего голосов;
   - `stars.USER` — оценка текущего пользователя (0, если не голосовал);
   - `average` — средневзвешенное по уровням, округление до 2 знаков;
   - `percent[level]` — доля голосов за уровень в процентах;
   - `starClasses` — массив из `maxLevel` элементов `full` | `half` | `empty` для отображения среднего (половина звезды при дробной части ≥ 0.5).
4. Заполняет `$this->arResult` текстами из конфига/параметров, пределами шкалы, цветами полос, блоком `FRONTEND_CONFIG` для JS.
5. Подключает шаблон.

### 5.2. `saveLevelAction($level)`

Публичный метод контроллера компонента (вызывается через `runComponentAction`).

**Фильтры** (`configureActions`):

- только HTTP POST;
- проверка CSRF (`ActionFilter\Csrf()`).

**Поведение:**

1. Неавторизованный пользователь (`GetID() <= 0`) — ошибка `AjaxJson::createError`, сообщение из конфига.
2. Пустой или нечисловой уровень — IM-уведомление (если включено), ошибка API.
3. Уровень вне диапазона `[getMinLevel(), getMaxLevel()]` — уведомление с «шутливым» текстом, ошибка API.
4. Иначе — обновление `USER[currentUserId]`, запись файла, уведомление об успехе, ответ с обновлённым блоком `widget` для фронта.

### 5.3. Уведомления (`notifyUser`)

Если параметр/конфиг `ENABLE_SYSTEM_NOTIFY` = `Y` и подключён модуль `im` с классом `CIMNotify`, пользователю отправляется системное уведомление intranet с префиксом `[ Название компании ]` в теле сообщения.

## 6. Конфигурация

### 6.1. `config.php`

Ассоциативный массив PHP. Основные группы ключей:

| Группа | Примеры ключей | Назначение |
|--------|----------------|------------|
| Пути и имя | `DATA_FILE`, `COMPONENT_NAME`, `SAVE_ACTION_NAME` | JSON и имена для AJAX |
| Шкала | `MIN_LEVEL`, `MAX_LEVEL`, `STAR_SYMBOL*`, `LEVEL_LABEL_SUFFIX`, `BAR_COLORS` | Диапазон оценок и оформление |
| Тексты UI | `FORM_TITLE`, `FORM_QUESTION`, `ANONYMITY_TEXT`, … | Заголовки, кнопки, плейсхолдер |
| Кнопка руководству | `ENABLE_MANAGEMENT_BUTTON`, `MANAGEMENT_BUTTON_TEXT`, `MANAGEMENT_URL` | Внешняя ссылка |
| Перенос DOM | `ENABLE_WIDGET_MOVE`, `WIDGET_MOVE_TARGET_ID`, `WIDGET_MOVE_POSITION`, `DOM_OBSERVER_TIMEOUT_MS` | Вставка виджета рядом с элементом страницы |
| Сообщения | `UI_MSG_*`, `API_ERROR_*`, `NOTIFY_MESSAGE_*` | Подсказки, ошибки API, тексты IM |

`COMPONENT_NAME` и путь к данным вычисляются от имени папок `dreamteam` и `level.happiness`, чтобы копирование компонента под другим кодом не ломало пути (при условии сохранения структуры каталогов).

### 6.2. `.parameters.php`

В визуальном редакторе доступны:

- **Путь к JSON файлу** (`DATA_FILE`) — по умолчанию из `config.php`;
- **Название компании** (`COMPANY_NAME`);
- **Отправлять системные уведомления** (`ENABLE_SYSTEM_NOTIFY`).

Остальные настройки задаются правкой `config.php` (или расширением `.parameters.php` при необходимости).

## 7. Фронтенд (`templates/.default/`)

### 7.1. `template.php`

- Корневой контейнер `#levelHappiness` с `data-component-name` и `data-save-action-name` для JS.
- Вывод средних звёзд, сводки, строк с барами по каждому уровню из `LEVEL_OPTIONS_DESC` (уровни от максимума к минимуму).
- Форма `#editLevel` с `bitrix_sessid_post()`, скрытое поле `userID` (для совместимости; фактическая отправка идёт через AJAX с `sessid`).
- Глобальный объект `window.LevelHappinessWidgetConfig` — сериализация `FRONTEND_CONFIG` через `CUtil::PhpToJSObject`.

### 7.2. `script.js`

- После загрузки DOM: опционально **перенос** виджета к элементу с id из конфига (по умолчанию `pulse_open_btn`) методом `insertAdjacentElement`. Если элемент ещё не в DOM, подключается `MutationObserver` на `document.body` с отключением по таймауту `domObserverTimeoutMs` (по умолчанию 5000 мс).
- Перехват `submit` формы: валидация уровня, вызов `BX.ajax.runComponentAction(componentName, saveActionName, { mode: 'class', data: { level, sessid } })`.
- Успех: `applyWidgetData` обновляет числа, ширины баров, звёзды, блок «ваш уровень»; показ сообщения через `BX.UI.Notification.Center` или встроенный `.lh-toast`.
- Защита от двойной отправки: флаг и блокировка кнопки на время запроса.

### 7.3. `style.css`

Карточка с градиентом, кастомные «звёзды» через псевдоэлемент `::before` и CSS-переменную ширины заливки (в т.ч. режим «половины» с поправкой `--lh-star-half-fill-width: 45%` из комментария в коде). Полосы процентов, стилизованный `<select>`, кнопки в блоке `.actions`.

## 8. Точка автоподключения (`init.php`)

Файл предназначен для подключения из `init.php` сайта или модуля: при корректном `$APPLICATION` вызывается `IncludeComponent` с именем из конфига, шаблоном по `WIDGET_STYLE`, без кеширования (`CACHE_TYPE` = `N`).

Использование: подключить путь к этому `init.php` в общем init-процессе портала, чтобы виджет появлялся на нужных страницах без ручной вставки в шаблон.

## 9. Безопасность и ограничения

- **CSRF**: действие `saveLevel` защищено фильтром ядра; форма содержит `sessid` для совместимости.
- **Авторизация**: сохранение только для вошедших пользователей.
- **Валидация уровня** на сервере и клиенте.
- **Конфиденциальность «анонимности»**: в интерфейсе заявлена анонимность статистики для коллег, но в JSON **хранится соответствие userId → оценка** — администратор с доступом к файлу может восстановить индивидуальные голоса. Для строгой анонимности потребовалась бы другая модель хранения (только агрегаты или отдельная политика доступа к файлу).

Права на запись: каталог `data/` должен быть доступен веб-серверу на запись, иначе `writeData` вернёт `false`.

## 10. Зависимости от модулей Bitrix

- Базовое ядро: компоненты, `Loader`, Engine (`Controllerable`, `ActionFilter`, `AjaxJson`, `Error`).
- Опционально: **`im`** — только для системных уведомлений; при отсутствии модуля виджет и сохранение работают, уведомления пропускаются.
- Для оценок **не выше порога** `LOW_SCORE_MAX_LEVEL` (по умолчанию 3): модули **`iblock`** и при необходимости **`lists`** — создание элемента универсального списка группы (`CIBlockElement::Add`) до записи JSON; при ошибке добавления оценка в файл не пишется.

## 11. Низкие оценки и список обращений

При выборе уровня от `MIN_LEVEL` до `LOW_SCORE_MAX_LEVEL` (по умолчанию 1–3) открывается модальное окно с обязательным текстом причины. AJAX `saveLevel` передаёт `message`; сервер проверяет непустой текст, затем создаёт строку списка с полями из `config.php` (`LIST_IBLOCK_TYPE_ID` = `lists_socnet`, `LIST_IBLOCK_ID` = 142, свойства `SOTRUDNIK`, `OTSENKA`, `SOOBSHCHENIE`, опционально `LIST_SOCNET_GROUP_ID` = 1). Только после успешного `Add` обновляется `levelHappiness.json`.

**Проверка на портале:** у пользователя должны быть права на добавление в список. Если `CIBlockElement::Add` возвращает ошибку из‑за формата поля «Сотрудник», в [`class.php`](class.php) в комментарии у `PROPERTY_VALUES` указан возможный вариант значения (`VALUE`); при необходимости скорректируйте формат под тип свойства в вашей версии коробки.

## 12. Ручное подключение в шаблоне

Пример (имя шаблона `.default` или кастомный при добавлении папки):

```php
$APPLICATION->IncludeComponent(
    'dreamteam:level.happiness',
    '.default',
    [
        'DATA_FILE' => '/local/components/dreamteam/level.happiness/data/levelHappiness.json',
        'COMPANY_NAME' => 'Команда Мечты',
        'ENABLE_SYSTEM_NOTIFY' => 'Y',
    ],
    false
);
```

Не забывать подключать JS/CSS шаблона (в Bitrix обычно через `$this->addExternalCss` / `addExternalJs` в `header.php` или автоматически при выводе компонента в зависимости от настроек шаблона сайта).

## 13. Резюме

Компонент **dreamteam:level.happiness** — это законченный виджет опроса удовлетворённости для портала на 1С-Битрикс с хранением в JSON, AJAX-обновлением статистики, настраиваемыми текстами и опциональными IM-уведомлениями. Код разделён на конфиг, класс с бизнес-логикой и шаблон с изолированными префиксами классов `lh-*`, что упрощает встраивание в существующую вёрстку.

Документ сгенерирован по состоянию файлов проекта; при изменении `config.php` или состава шаблонов имеет смысл актуализировать соответствующие разделы вручную.

## Реестр (`property122`) — правила отображения и использования

Источник: `property122.value` из `catalog.product.list/get`.

Файл с фактической вариативностью по всем разделам:
- `webhook_check/variability.report.pretty.json`

### Что показываем в UI
- **Лейбл**: всегда “Реестр”.
- **Значение**: строка `property122.value` как есть (trim).
- **Если пусто/`-`/null**: блок “Реестр” **не показываем**.

Примеры значений, которые реально встречаются (каталог `IBLOCK_ID=14`):
- `ФИС ФРДО`
- `Минтруд`
- `ЛНК. Независимый орган`
- `Ростехнадзор`
- `Реестр СРО`
- `Реестр МЧС`
- `Рособрнадзор`
- `Госгортехнадзор`
- `НРС. НОПРИЗ`
- `НРС. НОСТРОЙ`

### Что НЕ делаем
- Не заводим отдельное поле “ФИС ФРДО” — это **частный случай значения** реестра.
- Не меняем каталог Bitrix24 (правило read-only).

### Где используем в модуле
- `/academyprofi-catalog/products`: можно отдавать в каждом item (как `registry`) или оставить только для детальной (зависит от дизайна, но поле лёгкое).
- `/academyprofi-catalog/search`: не нужно.

Рекомендуемая форма в ответе API модуля:

```json
{
  "registry": "ФИС ФРДО"
}
```

### (Опционально) Нормализация/группировка для аналитики
Если потом понадобится группировать, делаем это **на стороне модуля**:
- `registryKey = lower(trim(registry))`
- никаких маппингов/переименований без требования — чтобы не потерять исходное значение.

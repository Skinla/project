# Инструкция для редакторов: блоки AcademyProfi (Поиск / Каталог / Деталка)

Эта инструкция для пользователей конструктора сайтов Bitrix24: как **добавить блок на страницу** и какие **настройки** доступны в блоках.

## Где найти блоки
- Открой страницу сайта в режиме редактирования (Конструктор сайтов).
- Нажми **Добавить блок**.
- В списке категорий ищи **`academyprofi`** (это категория наших блоков).
- Доступные блоки:
  - **Поиск услуг** (`academyprofi.search`)
  - **Каталог услуг** (`academyprofi.catalog`)
  - **Детальная карточка услуги** (`academyprofi.detail`)

## Общая логика (как всё работает вместе)
- **Поиск** и **Каталог** показывают услуги из каталога Bitrix по API.
- Переход в деталку идёт через hash в URL: `#product=<id>` (пример: `#product=8402`).
- **Деталка** на странице читает `#product=<id>` и показывает выбранную услугу. Если в URL нет `#product=...` — блок показывает заглушку “Выберите услугу в каталоге”.
- Для CRM‑формы дополнительно используется query‑параметр: `?productId=<id>` (он нужен, чтобы форма могла автоматически заполнить скрытое поле).
- **Цены**: отображаемая цена берётся из **цен каталога** Bitrix (`catalog.price.list`, `catalogGroupId=2`), а не из пользовательских свойств товара.

## 1) Блок “Поиск услуг” (`academyprofi.search`)

### Настройки (атрибуты блока)
Открой блок → **Настройки**:
- **ID каталога (IBLOCK_ID)** (`data-ap-iblock-id`)
  - Если оставить пустым — используется значение по умолчанию на сервере.
- **Путь detail-страницы (куда вести карточку)** (`data-ap-detail-path`)
  - Если заполнить (например `/okhrana_truda_detail/`) — результаты поиска будут открывать эту страницу: `<path>?productId=<id>#product=<id>`.
  - Если оставить пустым — используется дефолт (из настроек модуля) или остаёмся на текущей странице (меняем только hash).
- **ID папки каталога (раздел, iblockSectionId)** (`data-ap-section-id`)
  - Если указать — поиск ограничится одним разделом каталога.
  - Если пусто — поиск по всему каталогу.

### Контент (кнопка)
Открой блок → **Контент**:
- **Кнопка** — это редактируемый узел:
  - можно поменять **текст** (например “Найти” → “Искать”)
  - можно поменять **ссылку**

Рекомендация: если кнопка используется только для поиска, оставляй ссылку `#` (чтобы не было навигации).

### Дизайн
Открой блок → **Дизайн**:
- **Поле поиска**: фон, границы, радиус, padding слева/справа, цвет текста/hover, шрифт (family/size/weight)
- **Кнопка**: цвет/hover, радиус, padding, цвет текста/hover, шрифт (family/size/weight)
- **Фон/отступы блока**: стандартные настройки фона/паддингов для всего блока

## 2) Блок “Каталог услуг” (`academyprofi.catalog`)

### Настройки (атрибуты блока)
Открой блок → **Настройки**:
- **ID каталога (IBLOCK_ID)** (`data-ap-iblock-id`)
  - Пусто = берём дефолт на сервере.
- **Путь detail-страницы (куда вести карточку)** (`data-ap-detail-path`)
  - Если заполнить (например `/okhrana_truda_detail/`) — клик по карточке откроет эту страницу: `<path>?productId=<id>#product=<id>`.
  - Если пусто — остаёмся на текущей странице (меняем hash).
- **ID папки каталога (раздел, iblockSectionId)** (`data-ap-section-id`)
  - Ограничивает каталог одним разделом.
- **Колонки на десктопе (3 или 5)** (`data-ap-columns`)
  - `3 колонки` / `5 колонок`
- **Показывать срок оказания (раб.дни) вместо уч.часов** (`data-ap-show-duration-work-days`)
  - `Да` = в мета‑инфо карточки показываем **раб.дни**
  - `Нет` = показываем **уч.ч.**

### Дизайн
Открой блок → **Дизайн**:
- доступны стандартные настройки **фона/паддингов блока** (фон задаётся на контейнере блока).

## 3) Блок “Детальная карточка услуги” (`academyprofi.detail`)

### Как открыть услугу в деталке
Деталка выбирает услугу по URL:
- пример: `.../rabochie_professii_occg/#product=8402`

Обычно `#product=<id>` выставляется автоматически при клике по карточке в каталоге/поиске.

Если `#product=` нет — блок показывает заглушку и скрывает контент.

### Контент (CTA‑кнопка)
Открой блок → **Контент**:
- **Кнопка (CTA)** — редактируемый узел:
  - **текст кнопки** (например “Оставить заявку” → “Заказать”)
  - **ссылка кнопки**

Типовой сценарий: ссылка на якорь формы на этой же странице, например `#request`.

### Цена в UI
Цена “от … ₽” в карточках каталога/поиска/деталки берётся из **цен каталога** Bitrix:
- REST: `catalog.price.list`
- группа цен: `catalogGroupId=2`

Если нужно поменять группу цен (например, на другую розницу) — это задаётся конфигом модуля (ключ `bitrix24.catalogPriceGroupId`).

### Дизайн
Открой блок → **Дизайн**:
- **Кнопка (CTA)**: цвет/hover, радиус, padding, цвет текста/hover, шрифт (family/size/weight)
- **Фон/отступы блока**: стандартные настройки фона/паддингов для всего блока

## Рекомендованные сценарии настройки страниц
- **Страница раздела каталога** (одна “папка/направление”):
  - ставишь **Поиск** + **Каталог**
  - в обоих блоках указываешь одинаковый **ID папки (sectionId)**, чтобы выдача была внутри раздела
- **Отдельная страница деталки для каждого раздела (вариант B)**:
  - для каждого раздела создаёшь свою страницу деталки и ставишь туда блок **Детальная карточка услуги**
  - на странице раздела (где Поиск/Каталог) в обоих блоках заполняешь **“Путь detail-страницы”** своим путём (например `/okhrana_truda_detail/`)
- **Страница деталки**:
  - ставишь **Деталку**
  - ниже на странице ставишь форму и даёшь ей якорь `#request` (или другой), затем на CTA‑кнопке выставляешь ссылку на этот якорь

## Если “поменял настройки, а на сайте не видно”
- Проверь, что ты менял именно **узел кнопки** в “Контент”, а не только стили.
- Сделай жёсткую перезагрузку страницы (Ctrl+F5).
- Если изменения всё равно не применились, возможен кеш контента блока в Bitrix (редко): иногда помогает удалить блок со страницы и добавить его заново.

## Интеграция с CRM‑формой: передать `productId` в лид и прикрепить товар

### 1) Передать `productId` в лид (скрытое поле CRM‑формы)
Цель: чтобы в лиде заполнялось поле **`UF_CRM_1769508178701`** (productId).

Шаги в Bitrix24:
- CRM → Продажи → Каналы продаж → CRM‑формы → открыть нужную форму
- Включить **Expert mode**
- Вкладка **Hidden field values**
- Добавить поле лида **`UF_CRM_1769508178701`** и задать значение типа **Parameter** с именем параметра `productId`

После этого, когда пользователь откроет форму на странице с URL вида:
- `...?productId=8402#product=8402`
в лид автоматически улетит `UF_CRM_1769508178701 = 8402`.

### 2) Прикрепить товар к лиду (через БП/робота)
Цель: после создания лида прикрепить товар по `productId` (из `UF_CRM_1769508178701`) через продуктовые строки лида.

В БП/роботе делаем входящий webhook (одной строкой):

```text
https://akademia-profi.bitrix24.ru/rest/16/pw1kpzu5y54lb6ni/crm.lead.productrows.set.json?id={{ID}}&rows%5B0%5D%5BPRODUCT_ID%5D={{productId}}&rows%5B0%5D%5BQUANTITY%5D=1
```

Где:
- `{{ID}}` — ID лида из контекста БП/робота
- `{{productId}}` — значение поля `UF_CRM_1769508178701` из контекста БП/робота

Примечание:
- Метод `crm.lead.productrows.set` **перезаписывает** продуктовые строки лида. В нашей схеме предполагается, что в лидах из формы товаров нет, поэтому это безопасно.

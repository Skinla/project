## Проверка

**Коробка:** https://bitrix.milleniummedc.ru (не облако!)

1. **Карточка лида:** CRM → Лиды → открыть любой лид (например №55) → поле «Источник» должно показывать «Арбитражник 2».
2. **Справочники:** CRM → Настройки → Справочники → Источники — в списке должны быть UC_QUY0M1 и др. (внизу, SORT=500).
3. **Скрипт:** `ssh ... "cd /home/bitrix/www && php verify_sources_on_box.php"` — выводит все источники и примеры лидов.

Если не видно: обновите страницу (Ctrl+F5), проверьте что открыт именно коробка (bitrix.milleniummedc.ru).

## Почему в карточке лида не было новых источников (главная причина)

Список для поля «Источник» строится из `StatusTable::getStatusesList('SOURCE')` → ORM кэширует выборку из `b_crm_status` в **файлах** (`bitrix/cache`, `bitrix/managed_cache`, TTL ~1 ч).

Если источники добавлены **прямым SQL**, кэш не сбрасывается — в UI остаётся **старый набор** (например 14 строк без UC_*).

**Что делать:** после правок `b_crm_status` выполнить очистку кэша, например:

```bash
rm -rf /home/bitrix/www/bitrix/cache/* /home/bitrix/www/bitrix/managed_cache/*
```

Или из проекта: `php fix_source_cache_on_box.php`

Также для строк SOURCE задайте **`CATEGORY_ID = 0`** (как у штатных источников), не оставлять `NULL`.

## Про SORT

Имеет смысл держать SORT в разумных пределах (например 11–80), чтобы порядок в списке был логичным; отдельно от кэша это не блокировало показ — блокировал **устаревший ORM-кэш**.
